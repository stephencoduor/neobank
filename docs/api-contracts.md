## 1. API Conventions

### 1.1 Base URL

All endpoints are prefixed with:

```
https://api.neobank.co.ke/api/v1
```

### 1.2 Authentication

Every request (except `/auth/register`, `/auth/login`, `/auth/otp/*`) must include a Bearer JWT issued by Keycloak:

```
Authorization: Bearer <access_token>
```

Access tokens expire after 15 minutes. Use the refresh endpoint to obtain a new token pair.

### 1.3 Required Headers

| Header | Required | Description |
|---|---|---|
| `Authorization` | Yes (except public endpoints) | `Bearer <JWT>` |
| `Content-Type` | Yes (for request bodies) | `application/json` |
| `Accept` | Recommended | `application/json` |
| `X-Idempotency-Key` | Yes (financial POSTs) | UUID v4 -- prevents duplicate processing |
| `X-Request-Id` | Auto-generated by gateway | UUID v4 -- correlation ID for tracing |
| `X-Transaction-PIN` | Yes (financial operations) | Encrypted 6-digit PIN |
| `Accept-Language` | Optional | `en`, `sw`, `fr`, `am` (default: `en`) |

### 1.4 Pagination (Cursor-Based)

All list endpoints use cursor-based pagination:

**Request Parameters:**

| Param | Type | Default | Max | Description |
|---|---|---|---|---|
| `cursor` | string | - | - | Opaque cursor from previous response |
| `limit` | integer | 20 | 100 | Items per page |

**Response Envelope:**

```typescript
interface PaginatedResponse<T> {
  data: T[];
  cursor: {
    next: string | null;   // Pass as ?cursor= for next page
    prev: string | null;   // Pass as ?cursor= for previous page
  };
  total: number;            // Total matching items
}
```

### 1.5 Error Format

All errors follow a consistent structure:

```typescript
interface ApiError {
  code: string;             // Machine-readable error code (e.g., "ERR_INSUFFICIENT_FUNDS")
  message: string;          // Human-readable description
  details?: object;         // Additional context (field validation errors, etc.)
  traceId: string;          // X-Request-Id for support debugging
}
```

**Common HTTP Status Codes:**

| Code | Meaning |
|---|---|
| 200 | Success |
| 201 | Created |
| 400 | Bad Request -- validation error |
| 401 | Unauthorized -- missing or expired token |
| 403 | Forbidden -- insufficient permissions |
| 404 | Not Found |
| 409 | Conflict -- duplicate idempotency key or resource conflict |
| 422 | Unprocessable Entity -- business rule violation |
| 429 | Too Many Requests -- rate limit exceeded |
| 500 | Internal Server Error |

**Common Error Codes:**

| Code | HTTP | Description |
|---|---|---|
| `ERR_UNAUTHORIZED` | 401 | Missing or invalid JWT |
| `ERR_TOKEN_EXPIRED` | 401 | Access token has expired |
| `ERR_FORBIDDEN` | 403 | Insufficient role/permissions |
| `ERR_NOT_FOUND` | 404 | Resource not found |
| `ERR_VALIDATION` | 400 | Request body validation failed |
| `ERR_RATE_LIMITED` | 429 | Too many requests (100/min user, 20/min auth) |
| `ERR_IDEMPOTENCY_CONFLICT` | 409 | Same idempotency key with different payload |
| `ERR_INTERNAL` | 500 | Unexpected server error |

### 1.6 Date Format

All dates use ISO 8601 with timezone offset (East Africa Time):

```
2026-04-03T14:30:00+03:00
```

### 1.7 Currency & Money

Monetary amounts are always represented as integers in the smallest unit (cents) to avoid floating-point errors:

```typescript
interface Money {
  amount: number;       // Integer, smallest unit (e.g., 150000 = KES 1,500.00)
  currency: string;     // ISO 4217 code: "KES", "USD", "UGX", "TZS"
}
```

Supported currencies: `KES`, `USD`, `EUR`, `GBP`, `UGX`, `TZS`, `RWF`.

### 1.8 Rate Limiting

| Scope | Limit |
|---|---|
| General (per user) | 100 requests/minute |
| Auth endpoints | 20 requests/minute |
| Financial POST endpoints | 30 requests/minute |
| Admin endpoints | 200 requests/minute |

Rate limit headers on every response:

```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 87
X-RateLimit-Reset: 1712160000
```

## 2. Authentication -- `/api/v1/auth`

### 2.1 POST /auth/register

Start a new user registration (step 1 of 4: sends OTP to phone).

**Request:**

```typescript
interface RegisterRequest {
  phoneNumber: string;      // E.164 format: "+254712345678"
  countryCode: string;      // ISO 3166-1 alpha-2: "KE", "UG", "TZ", "RW", "ET"
  deviceId: string;         // UUID -- unique device identifier
  deviceType: string;       // "ANDROID" | "IOS" | "WEB"
  appVersion: string;       // Semantic version: "1.0.0"
}
```

**Response: 201 Created**

```typescript
interface RegisterResponse {
  data: {
    registrationId: string;    // "reg_01HXYZ..." -- used in subsequent steps
    otpSent: boolean;
    otpExpiresAt: string;      // ISO 8601
    otpLength: number;         // 6
  };
}
```

**Example:**

```json
// Request
POST /api/v1/auth/register
{
  "phoneNumber": "+254712345678",
  "countryCode": "KE",
  "deviceId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "deviceType": "ANDROID",
  "appVersion": "1.0.0"
}

// Response: 201 Created
{
  "data": {
    "registrationId": "reg_01HXY7K9M2N4P5Q6R8S0T1",
    "otpSent": true,
    "otpExpiresAt": "2026-04-03T14:35:00+03:00",
    "otpLength": 6
  }
}
```

**Error Codes:**

| Code | HTTP | Description |
|---|---|---|
| `ERR_PHONE_ALREADY_REGISTERED` | 409 | Phone number already has an account |
| `ERR_UNSUPPORTED_COUNTRY` | 400 | Country code not in supported list |
| `ERR_OTP_SEND_FAILED` | 502 | SMS gateway (Africa's Talking) unreachable |

### 2.2 POST /auth/login

Authenticate with phone number and PIN. Returns JWT + refresh token.

**Request:**

```typescript
interface LoginRequest {
  phoneNumber: string;      // "+254712345678"
  pin: string;              // 6-digit PIN (encrypted in transit)
  deviceId: string;         // Current device UUID
  deviceType: string;       // "ANDROID" | "IOS" | "WEB"
}
```

**Response: 200 OK**

```typescript
interface LoginResponse {
  data: {
    accessToken: string;       // JWT, 15-minute expiry
    refreshToken: string;      // Opaque token, 30-day expiry
    expiresIn: number;         // Seconds until access token expiry (900)
    tokenType: string;         // "Bearer"
    user: {
      id: string;              // "USR-001"
      firstName: string;
      lastName: string;
      phone: string;
      email: string | null;
      kycTier: number;         // 0=NONE, 1=BASIC, 2=STANDARD, 3=PREMIUM
      kycStatus: string;       // "none" | "pending" | "verified" | "rejected"
      tier: string;            // "Basic" | "Standard" | "Premium"
      createdAt: string;
    };
  };
}
```

**Example:**

```json

}

// Response: 200 OK
{
  "data": {
    "accessToken": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "dGhpcyBpcyBhIHJlZnJlc2ggdG9rZW4...",
    "expiresIn": 900,
    "tokenType": "Bearer",

      "id": "USR-001",
      "firstName": "Amina",
```

**Error Codes:**

| Code | HTTP | Description |
|---|---|---|
| `ERR_INVALID_CREDENTIALS` | 401 | Phone or PIN is incorrect |
| `ERR_ACCOUNT_LOCKED` | 423 | 5+ failed attempts -- 30-minute lockout |
| `ERR_ACCOUNT_FROZEN` | 423 | 10+ failed attempts -- requires support |
| `ERR_ACCOUNT_SUSPENDED` | 403 | Account suspended by admin |
| `ERR_DEVICE_NOT_BOUND` | 403 | Login from unrecognized device -- OTP required |

**Request:**

```typescript
interface KycStartRequest {
```

**Response: 201 Created**

```typescript
interface KycStartResponse {
  data: {
    verificationId: string;        // "KYC-001"
```

## 13. WebSocket Events

### 13.1 Connection

```
wss://api.neobank.co.ke/api/v1/ws?token=<access_token>
```

Authentication is performed via the `token` query parameter. The connection is closed if the token expires; the client should reconnect with a fresh token.

### 13.2 Message Format

```typescript
interface WebSocketMessage {
```

## Appendix B: Fineract Mapping Reference

Key mappings between NeoBank middleware endpoints and Apache Fineract REST API:

|---|---|---|

| `GET /accounts/:id/transactions` | `GET /fineract-provider/api/v1/savingsaccounts/{id}/transactions` | Paginated, cached |
| `POST /payments/send` (internal) | `POST /fineract-provider/api/v1/accounttransfers` | Between NeoBank accounts |
| `POST /payments/send` (M-Pesa) | `POST /fineract-provider/api/v1/savingsaccounts/{id}/transactions?command=withdrawal` | + M-Pesa B2C API |
| `POST /loans/apply` | `POST /fineract-provider/api/v1/loans` | + approve + disburse workflow |
| `GET /loans/:id/schedule` | `GET /fineract-provider/api/v1/loans/{id}/repaymentschedule` | Amortization table |
| `POST /loans/:id/repay` | `POST /fineract-provider/api/v1/loans/{id}/transactions?command=repayment` | |
| `POST /savings/goals` | `POST /fineract-provider/api/v1/fixeddepositaccounts` | Goals as fixed deposit sub-accounts |
| `POST /merchant/onboarding` | `POST /fineract-provider/api/v1/clients` | clientType: MERCHANT |
| `GET /admin/users/:id` | `GET /fineract-provider/api/v1/clients/{clientId}` | |
| `PUT /admin/users/:id/status` | `PUT /fineract-provider/api/v1/clients/{clientId}` | + activate/close commands |
